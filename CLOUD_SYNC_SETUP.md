# Googleスプレッドシート同期の設定

この手順では、Googleアカウント `hibiki.yashiro@gmail.com` に同期用スプレッドシートとApps Scriptを作成します。設定前や通信失敗時も、練習ログは従来どおり端末のlocalStorageへ保存されます。

## 1. スプレッドシートを作る

1. `hibiki.yashiro@gmail.com` でGoogleへログインします。
2. Googleスプレッドシートで空のファイルを作り、名前を `ゴルフ練習ログ` にします。
3. URLの `/d/` と `/edit` の間にある文字列を控えます。これが `SPREADSHEET_ID` です。

## 2. Apps Scriptを配置する

1. スプレッドシート上部の `拡張機能` → `Apps Script` を開きます。
2. エディタの `Code.gs` を削除し、このリポジトリの `apps-script/Code.gs` の内容を貼り付けます。
3. プロジェクト名を `Golf Practice Log Sync` に変更して保存します。

## 3. スクリプトプロパティを設定する

1. Apps Script左側の歯車アイコン `プロジェクトの設定` を開きます。
2. `スクリプト プロパティ` に次の2項目を追加します。

| プロパティ | 値 |
| --- | --- |
| `SPREADSHEET_ID` | 手順1で控えたID |
| `SYNC_TOKEN` | 自分で作った長いランダム文字列 |

トークンは32文字以上を推奨します。氏名、メールアドレス、パスワードは使わないでください。トークンはGoogleアカウントのパスワードではありません。

## 4. シートを準備する

1. Apps Script上部の関数選択で `setupSheets` を選びます。
2. `実行` を押します。
3. 初回だけGoogleの権限確認が表示されます。`hibiki.yashiro@gmail.com` を選び、スプレッドシートへのアクセスを許可します。
4. 元のスプレッドシートに `sessions` と `club_results` の2シートが作成されたことを確認します。

`sessions` は練習1回につき1行、`club_results` はクラブ1件につき1行です。可変入力項目を失わないよう、完全な内容を `payload_json` と `metrics_json` にも保存します。

## 5. Webアプリとしてデプロイする

1. Apps Script右上の `デプロイ` → `新しいデプロイ` を押します。
2. 種類は `ウェブアプリ` を選びます。
3. `次のユーザーとして実行` は `自分` を選びます。
4. `アクセスできるユーザー` は `全員` を選びます。
5. `デプロイ` を押し、表示された `/exec` で終わるURLを控えます。

公開URLからGoogle Apps Scriptへ直接通信するため、アクセス対象は `全員` が必要です。APIは共有トークンを検証しますが、静的Webアプリの仕組み上、完全な秘密認証にはなりません。URLを不用意に共有しないでください。

## 6. iPhoneでクラウド設定を保存する

1. 公開中のゴルフ練習ログをSafariで開きます。
2. `クラウド保存` → `クラウド設定` を開きます。
3. `Apps Script URL` に手順5のURLを入力します。
4. `共有トークン` に `SYNC_TOKEN` と同じ値を入力します。
5. `設定を保存` を押します。
6. `今すぐ同期` を押し、表示が `クラウド同期済み` になることを確認します。

設定値は端末のlocalStorageへ保存され、公開GitHubリポジトリには入りません。別のiPhoneやブラウザでは同じ設定をもう一度入力します。

## 7. 既存データを初回バックアップする

1. `既存データをクラウドへバックアップ` を押します。
2. 対象件数、同期済み件数、未同期件数を確認します。
3. 確認画面で続行します。

同じ練習IDとクラブ結果IDは更新扱いになるため、再実行しても二重登録されません。

## 動作確認

1. オンラインで新しい記録を保存し、端末保存後に `クラウド同期済み` になることを確認します。
2. スプレッドシートの両シートへ行が追加されることを確認します。
3. 機内モードで記録を保存し、`クラウド未同期` になることを確認します。
4. オンラインへ戻し、自動同期されることを確認します。
5. 別端末に同じURLとトークンを設定し、`今すぐ同期` で履歴が取り込まれることを確認します。
6. CSVが従来どおり出力できることを確認します。

## エラー時の確認ポイント

- `端末保存のみ`: Apps Script URLまたはトークンが未設定です。
- `同期エラー`: URL、トークン、Webアプリのアクセス設定、Apps Scriptの実行履歴を確認します。
- `Missing sheet`: Apps Scriptで `setupSheets` をもう一度実行します。
- `Invalid token`: Apps Scriptの `SYNC_TOKEN` とアプリへ入力した値を一致させます。
- タイムアウト: 記録は端末に残っています。通信状態を確認し、`今すぐ同期` を押します。

## Apps Scriptを更新したとき

1. `Code.gs` を更新して保存します。
2. `デプロイ` → `デプロイを管理` を開きます。
3. 対象デプロイの編集を開き、バージョンを `新バージョン` にします。
4. `デプロイ` を押します。

既存デプロイを更新すれば通常はURLを変えずに利用できます。新しいデプロイを作ってURLが変わった場合は、アプリのクラウド設定も更新してください。
